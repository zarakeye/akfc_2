# Authentication with tRPC

Dans ce tutoriel, nous allons comprendre comment l’authentification est organisée dans le projet.

L’objectif n’est pas seulement de lire quelques fichiers, mais de bien visualiser le rôle de chaque couche :

- le **modèle de données**
- la **validation**
- le **router tRPC**
- la **reconstruction de session**
- le **contexte serveur**

# 6 — Reconstruction de la session depuis la requête

C’est ici que le backend relit réellement l’état d’authentification.

<CodeExample
  language="ts"
  filepath="packages/backend/src/auth/getSessionFromRequest.ts"
  filename="getSessionFromRequest.ts"
  snippetRange="112-151"
  snippetHighlight="4-9,11-22,24,26"
  snippetTitle="Reconstruction de la session depuis la requête"
  fullFileTitle="Voir le fichier complet"
/>

Cette fonction suit une logique très claire :

1. lire le token
2. vérifier qu’il est valide
3. retrouver la session en base
4. vérifier qu’elle n’est pas expirée
5. transformer la session brute en session exploitable côté client

C’est une fonction centrale, car elle permet au backend de savoir si un utilisateur est déjà authentifié lors d’une requête suivante.

# 10 — Résumé du flux complet

Si on remet tout dans l’ordre :

1. le client envoie ses identifiants
2. le schéma Zod valide les données
3. le router tRPC déclenche la procédure `login`
4. le router délègue la logique métier à un service
5. une session est ensuite retrouvée ou reconstruite côté serveur
6. le contexte tRPC injecte cette session dans les requêtes suivantes
7. les procédures protégées peuvent alors s’appuyer sur cette identité

# 11 — Ce qu’il faut retenir

Voici les idées les plus importantes :

- **Zod** protège l’entrée
- **le router tRPC** orchestre la requête
- **le service métier** porte la logique de connexion
- **`getSessionFromRequest`** reconstruit l’état d’authentification
- **`createTRPCContext`** distribue cet état au reste du backend
- **`protectedProcedure`** bloque les requêtes non authentifiées

Autrement dit :

> l’authentification n’est pas une simple fonction login, mais une collaboration entre validation, router, session et contexte.
