# tRPC API Design

Dans ce tutoriel, nous allons voir comment **concevoir une API backend avec tRPC**.

tRPC permet de créer une API **entièrement typée entre le frontend et le backend**.

---

# Pourquoi tRPC

tRPC permet :

- typage end‑to‑end
- aucune génération de client
- intégration parfaite avec TypeScript

Comparaison :

| REST | tRPC |
|-----|-----|
| routes HTTP | procédures |
| types dupliqués | types partagés |
| client généré | client automatique |

---

# Structure typique

Dans le projet AKFC :

```
src/server/trpc
│
├ core
│   └ context.ts
│
├ routers
│   ├ auth.router.ts
│   ├ user.router.ts
│   └ course.router.ts
│
└ index.ts
```

---

# Création du router principal

```ts
export const appRouter = router({
  auth: authRouter,
  user: userRouter,
  course: courseRouter,
})
```

Ce router regroupe toutes les routes de l'API.

---

# Procédures

Une procédure est l'équivalent d'une route API.

Exemple :

```ts
export const courseRouter = router({
  getAll: publicProcedure.query(async () => {
    return prisma.course.findMany()
  })
})
```

---

# Validation avec Zod

tRPC fonctionne parfaitement avec **Zod**.

```ts
const createCourseSchema = z.object({
  title: z.string(),
  categoryId: z.string()
})
```

Puis :

```ts
create: protectedProcedure
  .input(createCourseSchema)
  .mutation(async ({ input }) => {
    return prisma.course.create({
      data: input
    })
  })
```

---

# Public vs Protected

Deux types de procédures sont utilisés.

### publicProcedure

Accessible sans authentification.

### protectedProcedure

Nécessite une session utilisateur.

---

# Context tRPC

Chaque requête possède un contexte.

```ts
export async function createTRPCContext(req) {
  const session = getSessionFromCookie(req)

  return {
    session
  }
}
```

# Utilisation côté frontend

Dans React :

```ts
const { data } = trpc.course.getAll.useQuery()
```

Pour mutation :

```ts
const mutation = trpc.course.create.useMutation()
```
