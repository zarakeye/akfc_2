# Learning Roadmap

Cette documentation n'est pas seulement une référence technique.
Elle constitue **un parcours pédagogique complet** à travers un projet full‑stack moderne.

L'objectif est double :

- comprendre **comment construire une application réelle**
- comprendre **pourquoi certaines décisions architecturales sont prises**

Le projet AKFC sert de **fil rouge** pour explorer les différentes couches d'une application moderne.

---

## Structure du parcours

La documentation est organisée en plusieurs grandes sections.

### Introduction

Cette partie explique :

- le contexte du projet
- les objectifs pédagogiques
- comment lire cette documentation

Fichiers principaux :

- `project-overview`
- `learning-roadmap`

---

### Foundation

La section Foundation pose les bases conceptuelles nécessaires pour comprendre le reste du projet.

On y explique :

- les principes de l'architecture choisie
- la logique du monorepo
- les conventions du projet

---

### Architecture

Cette section décrit **comment le projet est structuré**.

On y trouve notamment :

- l'architecture globale
- la stack technique
- l'architecture backend
- l'architecture frontend
- la préparation pour React Native

Ces chapitres permettent de comprendre **comment les différentes briques s'assemblent**.

---

### Domain

La section Domain décrit **le modèle métier**.

On y définit :

- les entités principales
- leurs relations
- leur rôle dans l'application

C'est ici que l'on pose les bases pour :

- la base de données
- l'API
- les formulaires frontend

---

### Tutorials

Les tutoriels constituent le cœur pédagogique de la documentation.

Chaque tutoriel explique :

1. le problème
2. l'approche architecturale
3. l'implémentation
4. le résultat

Exemples de tutoriels :

- authentification avec Next.js + tRPC
- modélisation Prisma
- gestion des formulaires
- gestion des médias Cloudinary
- construction d'un éditeur riche avec TipTap

---

### Patterns

Cette section explique certains **patterns utilisés dans le projet**.

Par exemple :

- DTO pattern
- Guards

- hooks personnalisés

### Infrastructure

Cette partie couvre les aspects plus opérationnels :

### Advanced

La section Advanced explore certains aspects plus poussés :

- moteur de documentation MDX

---

## Comment utiliser cette documentation

Si tu découvres le projet :

1. commence par **Project Overview**
2. lis ensuite **Architecture**
3. explore le **Domain Model**
4. attaque ensuite les **Tutorials**

Ce chemin te permet de comprendre progressivement le projet.

---

## Objectif final

À la fin de ce parcours, tu dois être capable de :

- comprendre l'architecture complète du projet
- reproduire certaines parties dans tes propres projets
- expliquer les choix techniques à d'autres développeurs

Autrement dit, cette documentation vise à transformer un projet personnel en **véritable support d'apprentissage**.
